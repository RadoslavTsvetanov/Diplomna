

### 1.2 Технологии, използвани в Diplomna

#### 1.2.1 Bun

Bun е ново Node.js runtime написано на Zig с страхотна производителност.

###### 1.2.1.2.1 Производителност и скорост

Bun е написан на Zig, който е много по-ефективен от V8 JavaScript engine на Node.js, предоставяйки значително по-добра производителност при стартиране и изпълнение.

###### 1.2.1.2.2 Вградени инструменти

За разлика от Node.js, Bun включва вградени инструменти като package manager и test runner, които елиминират нуждата от допълнителни инструменти като npm или Jest.

###### 1.2.1.2.3 По-добра DX (Developer Experience)

Bun предоставя по-добра developer experience с автоматична поддръжка на TypeScript, по-бързо инсталиране на пакети и по-ефективно горещо презареждане по време на разработка.

###### 1.2.1.2.4 Модерна архитектура

Bun е проектиран от нулата с фокус върху модерните JavaScript стандарти, включително ESM по подразбиране, което го прави по-съвременен избор от Node.js.

##### 1.2.1.3 Използване в Diplomna проекта

Използване в проекта: **Първична runtime** за всички пакети. Монорепото е оптимизирано за Bun, като се възползва от неговата скорост и вградени инструменти за по-ефективна разработка.

#### 1.2.2 TypeScript 5.0+

TypeScript е суперсет на JavaScript който добавя статични типи. Предимства:
- Пълна типозащита - поймане на грешки по време на компилация
- Интелиджънтен intellisense и автоматично допълване
- Лесна рефакторизация на код
- Документация чрез типове

Използване: **Всички пакети** в монорепото са написани на TypeScript със strict mode.


##### 1.2.2.1 Предимства на JavaScript като основа

Тъй като TypeScript е суперсет на JavaScript, той се възползва от следните предимства на JavaScript:

###### 1.2.2.1.1 Богата екосистема

JavaScript разполага с една от най-големите и най-активни екосистеми от библиотеки, инструменти и frameworks, което значително ускорява разработката и намалява нуждата от изграждане на решения „от нулата".

###### 1.2.2.1.2 Широко разпространено познание

Езикът е добре познат на голяма част от разработчиците, което улеснява onboarding-а на нови членове в екипа и намалява бариерата за участие в проекта.

###### 1.2.2.1.3 Високо ниво на абстракция

Като език от по-високо ниво, JavaScript позволява по-бързо и по-изразително писане на код в сравнение с по-нисконивови езици като C++, което повишава продуктивността.

###### 1.2.2.1.4 Де факто стандарт за full-stack уеб разработка

JavaScript е утвърденият избор за разработка както на frontend, така и на backend приложения, което позволява използването на един и същ език в целия технологичен стек.

##### 1.2.2.2 Защо TypeScript над JavaScript

###### 1.2.2.2.1 Типова безопасност върху JavaScript

TypeScript надгражда JavaScript, като добавя статична типизация, която позволява улавяне на голяма част от грешките още по време на компилация.

###### 1.2.2.2.2 Абстракции без runtime цена

Типовата система на TypeScript съществува единствено по време на компилация и не добавя допълнителни runtime разходи, което го прави ефективен избор от гледна точка на производителност.

###### 1.2.2.2.3 Мощни типови трансформации по време на LSP анализ

TypeScript позволява сложни типови трансформации и проверки още на ниво language server (LSP), което в много случаи елиминира нуждата от допълнителна code generation логика — възможност, рядко срещана в други езици.

###### 1.2.2.2.4 Ползи дори при използване на JavaScript стил

Дори когато кодът е написан в стил, близък до чист JavaScript, TypeScript предоставя значителни предимства благодарение на своята силна система за type inference.

###### 1.2.2.2.5 Постепенно въвеждане на типова строгост

TypeScript позволява плавно преминаване по „спектъра на типовата безопасност“, като дава възможност кодовата база да се изчиства и подобрява постепенно, без нужда от рязка и рискова миграция.
